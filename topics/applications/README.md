# Applications

This category contains documentation for self-hosted applications, web applications, and software platforms.

## Current Topics

- **[Coolify](coolify/README.md)** - Self-hosted deployment platform alternative to Heroku/Netlify/Vercel

## What Belongs Here

Applications are software platforms that users deploy, install, or run. Examples include:
- Self-hosted platforms (e.g., Coolify, Nextcloud, GitLab)
- Web applications (e.g., WordPress, Ghost, Discourse)
- Desktop applications (e.g., VS Code, IntelliJ IDEA)
- Server applications (e.g., Nginx, Apache, PostgreSQL)
- Container platforms (e.g., Portainer, Rancher)

## Adding a New Application

1. Create a directory for your application: `mkdir <app-name>`
2. Copy the template: `cp ../../.templates/topic-template.md <app-name>/README.md`
3. Fill in all relevant sections
4. Update this README to list your application
5. Update the main repository README and INDEX

## Template Sections to Emphasize

For applications, pay special attention to:
- **Installation/Setup**: Detailed installation instructions
- **Requirements**: System requirements and dependencies
- **Common Commands**: How to start, stop, configure the application
- **Use Cases**: Real-world scenarios where this application is useful
