## What Belongs Here

Frameworks are structured libraries and tools that provide a foundation for building applications. Examples include:
- Web frameworks (e.g., React, Vue, Angular, Django, Rails, Express)
- Mobile frameworks (e.g., React Native, Flutter, Ionic)
- Backend frameworks (e.g., Spring, Laravel, FastAPI, NestJS)
- Testing frameworks (e.g., Jest, PyTest, JUnit)
- UI frameworks (e.g., Bootstrap, Tailwind, Material-UI)

## Adding a New Framework

1. Create a directory for your framework: `mkdir <framework-name>`
2. Copy the template: `cp ../../.templates/topic-template.md <framework-name>/README.md`
3. Fill in all relevant sections
4. Update this README to list your framework
5. Update the main repository README and INDEX

## Template Sections to Emphasize

For frameworks, pay special attention to:
- **Key Features**: What makes this framework unique
- **Architecture**: How the framework structures applications
- **Common Commands/Usage**: Project setup, development, and build commands
- **Best Practices**: Recommended patterns and conventions
- **Use Cases**: When to choose this framework over alternatives
