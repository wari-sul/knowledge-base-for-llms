## What Belongs Here

Tools are utilities that assist developers in building, testing, deploying, and maintaining software. Examples include:
- Version control (e.g., Git, Mercurial)
- Containerization (e.g., Docker, Podman)
- Orchestration (e.g., Kubernetes, Docker Compose)
- Build tools (e.g., Webpack, Vite, Gradle, Make)
- Package managers (e.g., npm, pip, cargo, Maven)
- CI/CD tools (e.g., Jenkins, GitHub Actions, GitLab CI)
- Monitoring tools (e.g., Prometheus, Grafana)
- Database tools (e.g., DBeaver, pgAdmin)

## Adding a New Tool

1. Create a directory for your tool: `mkdir <tool-name>`
2. Copy the template: `cp ../../.templates/topic-template.md <tool-name>/README.md`
3. Fill in all relevant sections
4. Update this README to list your tool
5. Update the main repository README and INDEX

## Template Sections to Emphasize

For tools, pay special attention to:
- **Installation/Setup**: How to install and configure the tool
- **Common Commands/Usage**: Practical examples of everyday usage
- **Use Cases**: Problems this tool solves
- **Best Practices**: How to use the tool effectively
- **Common Pitfalls**: Mistakes to avoid
